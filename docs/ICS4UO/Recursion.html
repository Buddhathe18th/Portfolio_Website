Recursion

Imagine a scenario, you want to teach someone how to do factorials, specifically what 5! is, but you forgot the formula. Confused, you go ask your friend Albert. He doesn’t know what the value is, but he knows 5! is equal to 5*4!. Then, you find your friend Bernie, who doesn’t know the value of 4!, but knows 4!=4*3!. Then your friends Catherine, and Danielle know 3!=3*2! and 2!=2*1! Respectively. Then you ask your last friend, Earl, who tells you 1!=1. Then, with that information, you can find 2!, which helps you find 3! and so on up until 5!, which you find is 120. During this process, you never find out how to calculate factorials, but you can now find the values of all the factorials, which this process is called recursion. 

public int factorial(int n) {
    return n*factorial(n-1);
}
If you like to look at some code, the process we had above with the friends can be modelled with this code. Each method call is equal to a time you ask a friend what a factorial is equal to.

So what is recursion exactly? Recursion is when a method is defined with a call to itself. For example, in our factorial adventure, we have a method called factorial(), which takes a positive integer parameter n and returns the factorial of n. To use recursion, we can use the fact that n!=n*(n-1)!, so what we do is return n*factorial(n-1); Now you may ask, when does the calling of the methods stop? Will it ever stop? In the code we just wrote, it will never end, because it will continuously multiply integers lower and lower until it either reaches past the smallest value for integers or overflows the stack. That’s why in recursion, we need something called a base case.

Base cases are the cases that we know about, the ones that we have a definite solution for. For the factorial method, our base case is that 1!=1. So in our method, before we return, we can add an if statement: 

public int factorial(int n) {
    if (n==1) {
        return 1;
    }
    return n*factorial(n-1);
}

So when the original n goes down and down, all the way until 1, it will return a value, and it returns all the way to the first method call, giving us the desired value of n!.

Why is Recursion Important?

Sometimes, problems can be very long and complex, making them very hard to tackle with normal methods. With recursion, we solve the hard problem by a definition of an easier part and go on continuously until we reach a case that we know the answer to.

Downside of Recursion

Recursion can be a great tool for tackling problems, but when you use it, make sure you have a base case. If a recursive method never reaches a base case, it will infinitely make method calls. All the data used in each method call is kept on a data structure called the stack. When you call too many methods, the stack will run out of space and a data leak called stack overflowcan happen, in which the data you meant to write to the stack is written somewhere else onto another piece of memory.

Where is Recursion Used?

Recursion is commonly used in searches and sorts. This is because searches and sorts generally are built up from the idea of sorting each small piece, then putting all the sorted pieces back together, or subdividing an array until the location of the wanted element is found. All of this will be covered in sorting and searching.
